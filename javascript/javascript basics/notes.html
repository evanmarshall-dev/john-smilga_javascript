JavaScript Basics

Implicit Type Conversion

What if we perform a math function on a string?

For example if you subtract a string from another it will say NaN error however if you subtract a string with a number in it ("23") from another it will still perform the calculation because JS will convert the string number into a number type. If you add two number strings they will concatenate as a string type. Even if you add a number string to number type it will still concatenate as a string type. This commonly happens when a user inputs a number in number input and you do not convert the value of the inputs to correct type.

Data Types

There are seven data types. Including primitive and all else are objects. Primitive (string, number, boolean, null, undefined and symbol (ES6)). Objects (arrays, functions and objects). You can always console.log "typeof" to determine the type of a value. Null will come back as typeof object even though it should be null (this is a bug since a lot of code relies on null).
